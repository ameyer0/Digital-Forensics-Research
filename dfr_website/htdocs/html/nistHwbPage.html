Hardware Write Blockers

An evaluation of test results for the hardware write block tool Tableau that also includes a hands-on activity with the purpose of becoming familiar with a hardware write blocker and its basic functionality.

A. Test Case and Test Drive Description

The purpose of the experiments was to test a write blocker’s ability to write-protect a SATA drive. This test can be repeated to test multiple types of connections (interfaces) between a computer and the write blocker. Test the ability of the write blocker to block write commands from the ATA and SCSI command sets issued from a test computer from modifying a SATA drive. The manufacturer, model & size of the test drive used for this test: The hard drive as a 160gb drive manufactured by western digital.


B. HWB Evaluation Module: Exercise for Students

1. Goals:

To become familiar with (a) Hardware Write Blocker (HWB) and its basic functionality, (b) NIST CFTT Federated Testing system, and how to use this system to evaluate a HWB.


2. Preparation

Visit NIST CFTT program’s website on HWB, and read documents on HWB specifications Version 2.0 and the test plan for Version 2.0. here.


3. Tasks

The first task is to use the Federated Testing iso file (which is available in C: drive in any computer in lab) to start a VM on VirtualBox. Below are some of the intermediate steps (i.e. screenshots), which you will go through.


As for the second task; On the welcome page of Federated Testing Forensic Tool Testing Environment, click on “Test a hardware write block tool” (which is available on the left panel). Then, walk over the following steps to test a HWB:

  1. Go to “Hardware Write Block Home” page and click on “ … Get Started”.
  2. Now click on “Format your Log Drive”. Assuming you do not use an external flash drive, system will show internal virtual drive /dev/sda to you to select as the log drive.
  3. Follow the instruction to mount the Log Drive.
  4. Go to “Generate test cases and start testing”.
  5. Select a hardware write block type. Choose the “hard drive” option.
  6. Describe the write blocker. Put in detailed information on manufacturer and model name, and so on. Select the drive type as SATA. Choose USB3 as the type of connection between test computer and HWB.
  7. Review test selections.
  8. Select the FT-HWB-SATA option and ‘Run test case’.
  9. Get results.
  10. Generate a test report; a test report copy will be written in the Log Drive.
  11. Open the test report in a Windows computer, and copy the content of the report to a new word document. This will look like the following.

Now delete or replace the green text with related, correct, or specific information. As an example, under “2.Testing Organization” write the following.
Organization conducting test: BGSU
Contact: S. Roy (sanroy@bgsu.edu)
Report date: Nov 14, 2019
Authored by: your name
Reviewed by: other members in your group
Reviewed by date: Nov 14, 2019

  1. Go to the log file in the Log Drive (something like hwbtestlog.txt) and find the detailed test results. Now focus on a specific write command (e.g., opcode 30h). During the test, this command (30h) was issued to write to one sector (say S) of the hard disk, and the details are in the appendix. Check the (reported) content of the sector S before the test and after the test. Briefly verify whether they are “unchanged”.
  2. Submit your test-report on Canvas.
